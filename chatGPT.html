Chat.OpenAI
1. Models/Engine:
1.1. Davinci (text-davinci-003)
1.1.1. to comprehending text and generating responses that are more nuanced like summarizing for a child or emulating human speaking patterns
1.1.2. Good at: Complex intent, cause and effect, summarization for audience
1.1.3. Max Token = 4000
1.2. Curie (text-curie-001)
1.2.1. capable of analyzing text, answering direct questions, and providing key points
1.2.2. Good at: Language translation, complex classification, text sentiment, summarization
1.2.3. Max Token = 2048
1.3. Babbage (text-babbage-001)
1.3.1. Good at: Moderate classification, semantic search classification
1.3.2. Idea iteration
1.3.3. Sentence completion
1.3.4. Plot generation
1.3.5. Max Token = 2048
1.4. Ada (text-ada-001)
1.4.1. Good at: Parsing text, simple classification, address correction, keywords
1.4.2. Random data
1.4.3. Character descriptions
1.4.4. Max Token = 2048
1.5. Codex
1.5.1. code-davinci-002
1.5.1.1. Most capable Codex model. Particularly good at translating natural language to code. In addition to completing code, also supports inserting completions within code.
1.5.1.2. Max Token = 8000
1.5.2. code-cushman-001
1.5.2.1. Almost as capable as Davinci Codex, but slightly faster. This speed advantage may make it preferable for real-time applications.
1.5.2.2. Max Token = 2048
2. Variable need to set
2.1. Model
2.1.1. Engine atau model, ada 6 jenis yang digunakan
2.2. Prompt
2.2.1. Input yang akan diperintahkan pada AI. Berikan input/contoh dan tambahkan instruksi.
2.3. Max Tokens
2.3.1. Token ditentukan dari satu kata atau potongan kalimat. 1 Token antara 2 - 4 text, tergantung jenis bahasa yang digunakan. Variabel yang dimasukkan berupa angka (integer). Berapa Mak Token yang akan Anda gunakan pada responnya nanti.
2.4. Temperature
2.4.1. Tentukan angka dari 0 - 1 (angka paling kecil agar mesin tidak perlu banyak mikir kreative atau cukup yes no). Pilih gunakan Temperature atau Top P, test di playground atau cek penjelasan di How To
2.5. Top P
2.5.1. Variable standar angka 1. Memiliki fungsi yang sama seperti Temperature. Gunakan angka 1 jika salah satunya tidak digunakan
2.6. Frequency Penalty
2.6.1. frequency penalty is bigger if the token has appeared multiple times
2.7. Presence Penalty
2.7.1. presence penalty is a flat reduction if the token has appeared at least once before
2.8. Stop Sequence
2.8.1. Untuk membatasi respon yang dihasilkan. Terutama untuk respon berupa daftar. Variabel yang dimasukkan berupa angka atau kalimat
3. What for?
3.1. Language/Text Completion - pakai GPT-3
3.2. Code Completion - pakai Codex
3.3. Image Generation - pakai DALL-E
4. Things you should remember
4.1. Pilih jenis engine sesuai kebutuhan
4.2. Rata-rata setiap engine menggunakan sekitar 2048 Token atau sekitar 1500 kata dalam bahasa inggris. Setiap mesin memiliki maks token yang dapat digunakan. Lihat di penjelasan Engine/Model
4.3. Token dihitung sesuai kata pada responnya (result)
4.4. Untuk mendapatkan hasil AI yang baik, berikan contoh pada perintahnya
4.5. Gunakan kalimat perintah yang jelas dan tegas
4.6. Untuk hasil AI terbaik, gunakan kalimat perintah dalam bahasa Inggris (OpenAI menggunakan data latihan input dalam bahasa inggris)
4.7. Respon AI menggunakan data hingga bulan Juni 2021
4.8. Perhatikan Rate Limit. Ada batasan berapa kali Anda bisa menggunakan API dalam kurun waktu tertentu. Pembatasan ditentukan oleh penggunaan Request atau Token, mana yang lebih dahulu dicapai.
4.9. Jangan tambahkan 'spasi" pada akhir perintah, karena akan membuat bingung AI dalam memberikan respon

5.7. Make AI behave
5.7.1. Top P
5.7.1.1. Top P provides better control for applications in which GPT-3 is expected to generate text with accuracy and correctness
5.7.2. Temperature
5.7.2.1. Temperature works best for those applications in which original, creative or even amusing responses are sought.
